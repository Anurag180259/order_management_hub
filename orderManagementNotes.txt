queryParam
	During some operations if we need some details it can be send through query param, but query param once defined can be optional, and this is not
	the thing that we are working on there is something that the request is working on but there are some details that we need for this initial thing to 
	work
		example:if we are fetching weather data of the specific city the city name become the resource that we are working on
				but the time of the day becomes a parameter needed to perform the main operations
				so time of the day can be sent as the query param, or a date can be send using same
pathParam
	This is the parameter that we are actually working upon,
		example:in this very project while restocking, we are acting upon a specific resourse that is a specific product, so the product id will
				be sent in the path param
In short pathParam identifies the specific resource that we are working on and query param is non mandatory detail required for performing action on the main resource
